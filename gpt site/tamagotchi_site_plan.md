# 웹 다마고치 서비스 사이트 기획 정리

## 1. 프로젝트 방향 요약

이 사이트는 단순한 소개 페이지가 아니라, **게임을 중심으로 소개·소식·커뮤니티·마이페이지가 연결된 하나의 서비스**로 설계하는 것이 적합하다.

핵심 방향은 다음과 같다.

- 소개 사이트와 게임 사이트를 분리하지 않는다.
- **홈 → 플레이 → 커뮤니티/소식 → 마이페이지**로 이어지는 순환 구조를 만든다.
- 모바일에서는 **게임만 몰입해서 볼 수 있는 전용 플레이 화면**을 제공한다.
- 디자인은 너무 SaaS 같지도, 너무 유치한 아동용 사이트 같지도 않게 **따뜻한 디지털 펫 감성**으로 간다.

## 2. 전체 서비스 구조

### 2.1 추천 최상위 구조

```txt

├─ /guide            시작 가이드 / FAQ

├─ /community        게시판

│  └─ /suggestions   건의
├─ /me               마이페이지
```

- 홈
- 플레이
- 커뮤니티
- 소식
- 마이

#### 모바일 하단 메뉴
- 홈

`가이드`, `FAQ`, `정책`, `문의`는 메인 탭까지 올리지 않고, 홈/푸터/마이페이지 내부에서 접근하도록 하는 것이 좋다.

## 3. 게임 화면 구성 전략

### 3.1 기본 방향

게임 화면은 서비스 안에 자연스럽게 포함시키는 것이 좋다.

우선순위는 다음과 같다.

1. `/play` 페이지에서 게임을 직접 띄운다.
2. 독립 빌드가 있다면 같은 도메인 내 `/game/` 자산을 `/play`에서 same-origin iframe으로 감싼다.
3. 별도 도메인 cross-origin iframe 방식은 초기에 피하는 것이 좋다.

### 3.2 이유

- 로그인/저장/세션 유지가 훨씬 안정적이다.
- 브라우저의 same-origin 제약, third-party cookie 제한, iframe 정책 이슈를 줄일 수 있다.
- 운영 복잡도가 낮고, 게임과 커뮤니티/공지/마이페이지를 하나의 서비스 흐름으로 연결하기 쉽다.

## 4. 모바일 플레이 전략

### 4.1 권장 방식

모바일에서는 **새창 중심이 아니라, 전용 게임 화면 중심**으로 가는 것이 좋다.

권장 흐름:

- 모바일에서 `게임 시작` 클릭
- `/play/full` 진입
- 헤더/푸터/게시판 링크 제거
- 게임만 꽉 차게 보여줌
- 작은 오버레이 버튼만 유지

### 4.2 버튼 구성

`/play/full` 에는 아래 정도만 남기는 것을 추천한다.

- 돌아가기
- 전체화면
- 설치 안내

### 4.3 새창/새탭은 보조 옵션

새창이나 새탭 열기는 선택 사항으로 둘 수는 있지만, 기본 진입 방식으로 삼는 것은 비추천이다.

이유:
- 모바일 브라우저에서 새창 UX가 일관되지 않다.
- 팝업 차단에 걸릴 수 있다.
- 사용자 입장에서 맥락 전환이 발생한다.

### 4.4 PWA 방향

최종적으로는 **PWA 설치 경험**을 붙이는 것이 가장 이상적이다.

권장 순서:
1. `/play/full` 전용 화면 확보
2. `standalone` 중심의 설치 경험 제공
3. 필요 시 fullscreen은 보조 옵션으로 제공

### 4.5 결론

핵심 전략은 아래 한 줄로 정리된다.

> **새창은 옵션, 전용 게임 화면은 필수, PWA는 완성형**

## 5. 페이지별 역할

### 5.1 홈 `/`

홈은 단순 설명 페이지가 아니라 **첫인상 + 복귀 허브**다.

#### 비로그인 홈 역할
- 이 게임이 무엇인지 3초 안에 이해
- 대표 비주얼로 감정적 훅 제공
- `지금 키우기` CTA 제공
- 핵심 재미를 간단히 설명
- 최신 소식과 커뮤니티 분위기를 보여줌

#### 로그인 홈 역할
- 내 펫 상태 요약
- 오늘 할 일 안내
- 바로 이어하기 제공
- 새 공지/새 댓글/보상 등 복귀 동기 제공

### 5.2 플레이 `/play`
- 사이트 안의 기본 플레이 진입 화면
- 데스크톱 중심 기본 플레이 화면
- 최소한의 상단 액션만 유지
- `전체화면`, `소리`, `돌아가기` 정도만 제공

### 5.3 몰입형 플레이 `/play/full`
- 모바일 전용 핵심 화면
- 헤더/푸터 제거
- 게임만 꽉 차게
- 아주 작은 오버레이 버튼만 제공

### 5.4 가이드 `/guide`
- 시작 방법
- 성장 방법
- 아이템 설명
- FAQ
- 저장/로그인 문제 해결

### 5.5 소식 `/news`
- 공지
- 패치노트
- 이벤트
- 점검 안내

커뮤니티와 운영 공지는 반드시 분리하는 것이 좋다.

### 5.6 커뮤니티 `/community`
초기 추천 카테고리:
- 내 펫 자랑
- 자유게시판
- 질문
- 버그 제보
- 건의

이 장르는 경쟁보다 **애착 공유**가 잘 작동하므로 `내 펫 자랑`을 가장 앞에 두는 것이 좋다.

### 5.7 마이페이지 `/me`
마이페이지는 단순 설정 화면이 아니라 **내 펫 허브**로 잡는다.

구성 예시:
- 내 펫 상태
- 최근 플레이
- 업적/도감
- 아이템 보관함
- 알림 설정
- 계정 설정

### 5.8 지원 `/support`
- 문의
- 신고
- 이용약관
- 개인정보처리방침
- 계정/저장 도움말

## 6. 정보구조와 운영 원칙

### 6.1 구조 원칙
- 구조는 단순하게 유지한다.
- 소개성 페이지와 커뮤니티, 마이페이지가 게임을 보조하는 형태로 배치된다.
- 메인 이동축은 좁게 유지하고, 나머지는 내부 진입형으로 설계한다.

### 6.2 커뮤니티 운영 원칙
초기에는 무거운 포럼보다 **심플 게시판**이 적합하다.

추천 카테고리:
- 공지
- 자유
- 버그/문의
- 자랑/스크린샷
- 건의

## 7. 디자인 톤앤매너 제안

### 7.1 핵심 콘셉트

추천 콘셉트:

> **Cozy Digital Pet**

또는

> **Cozy Retro Device**

정리하면,

- 귀엽지만 유치하지 않게
- 따뜻하지만 흐릿하지 않게
- 앱처럼 정돈되어 있지만 감정은 살아 있게

즉,

> **정돈된 앱 UI 위에 디지털 장난감 감성을 얹는 방향**

### 7.2 톤의 비율

추천 비율:
- **정돈된 제품 UI 70~80%**
- **캐릭터성과 감정 표현 20~30%**

서비스 영역(공지, 게시판, 설정)은 더 차분하고,
브랜드 경험 영역(홈, 플레이 진입, 마이페이지, 보상 순간)은 더 따뜻하고 감정적으로 간다.

## 8. 비주얼 스타일 가이드 초안

### 8.1 컬러 방향

추천 무드:
- 배경: 따뜻한 아이보리
- 표면: 크림 화이트

- 주색: 민트/세이지
- 보조색: 살구
- 포인트색: 라벤더


### 8.2 시각적 질감
- 둥근 카드

- 부드러운 여백
- 배지는 스티커/도장 느낌
- 픽셀 요소는 포인트로만 제한 사용

## 9. 컴포넌트 원칙

### 9.1 버튼
- 메인 CTA는 크고 넓게
- 모바일에서는 텍스트 링크보다 버튼 중심
- `플레이`, `밥 주기`, `청소`, `글쓰기` 같은 액션은 엄지로 누르기 쉽게 크게 설계

### 9.2 상태 표시
펫 상태는 숫자만 보여주지 않고 아래를 함께 쓴다.

- 아이콘
- 텍스트
- 상태 바
- 필요 시 표정/짧은 모션

예시:
- 배고픔: `보통`
- 기분: `조금 심심해요`
- 청결: `아주 깨끗해요`

### 9.3 카드
홈과 마이페이지는 카드형 정보 구조가 잘 맞는다.

예시 카드:
- 내 펫 카드
- 오늘 할 일 카드
- 새 소식 카드
- 인기 글 카드

## 10. 모션 원칙

좋은 모션:
- 펫이 숨 쉬듯 살짝 움직임
- 알림 배지가 통통 튐
- 버튼 탭 시 짧은 반응
- 도감 해금/보상 시 짧은 축하 연출

피해야 할 모션:
- 과한 패럴랙스
- 큰 줌 인/아웃
- 전체 화면 흔들림
- 긴 로딩 애니메이션

핵심은 **살아 있다는 느낌만 주는 것**이다.

## 11. 성능 관점

이 서비스는 예쁜 것보다 먼저 **빠르게 들어가서 내 펫을 확인할 수 있는 느낌**이 중요하다.

따라서 피해야 할 것:
- 무거운 자동재생 영상
- 과한 3D 배경
- 지나친 글래스모피즘
- 과도한 떠다니는 장식

핵심 키워드:
- 빠른 첫 진입
- 즉시 반응성
- 가벼운 인터랙션

### 비로그인 홈 의도
첫 화면에서 사용자가 바로 이해해야 하는 것은 4가지다.

1. 무슨 게임인지
2. 지금 눌러야 할 버튼이 뭔지
3. 살아 있는 서비스인지
4. 들어가면 뭐가 재밌는지

즉, 첫 화면에서는 **대표 비주얼 + 한 줄 설명 + 큰 CTA**가 가장 먼저 보여야 한다.

로그인 홈은 홍보가 아니라 **복귀와 유지율**이 목적이다.

우선순위는 다음과 같다.
- 내 펫
- 오늘 할 일
- 바로 이어하기
- 새 소식
- 커뮤니티

## 13. 홈에서 꼭 들어가야 하는 블록

### 13.1 히어로 카드
구성 요소:
- 대표 캐릭터 또는 알
- 한 줄 설명
- 큰 CTA 버튼
- 보조 버튼 1개
- 짧은 신뢰 문구
  - 예: `웹에서 바로 플레이`, `저장 연동`, `모바일 몰입 모드`

핵심은 **귀여움보다 행동 유도**다.

### 13.2 내 펫/복귀 카드
구성 요소:
- 펫 비주얼
- 이름
- 레벨
- 상태 3개
- 이어하기 버튼

### 13.3 소식 블록
구성 요소:
- 카테고리 배지
- 제목
- 날짜
- 1줄 요약

### 13.4 커뮤니티 하이라이트
우선순위는 일반 자유글보다 다음 콘텐츠가 앞선다.
- 내 펫 자랑
- 스크린샷
- 성장 기록

## 14. 홈 카피 예시

### 14.1 히어로 문구
- **하루 1분, 작은 펫을 키워요**
- 먹이고, 돌보고, 진화시키고, 매일 조금씩 가까워져요.

버튼:
- **지금 키우기**
- 게임 소개

### 14.2 로그인 상태 상단 문구
- **모찌가 오늘도 기다리고 있어요**
- 잠깐 들어가서 상태를 확인해볼까요?

### 14.3 가이드 블록 문구
- **처음이라면 여기서 1분만**
- 펫 만들기부터 저장까지 한 번에 익혀요.

## 15. 홈에서 빼는 것이 좋은 것

- 긴 세계관 설명
- 너무 많은 배너
- 첫 화면 전체를 차지하는 공지
- 과한 애니메이션
- 너무 많은 퀵메뉴
- 게시판 최신글 과다 노출

홈은 **모든 것을 설명하는 페이지가 아니라, 가장 중요한 행동으로 보내는 페이지**여야 한다.

## 16. 현재까지의 추천안 요약

### 구조
- 홈
- 플레이
- 플레이 전체화면
- 가이드
- 소식
- 커뮤니티
- 마이페이지
- 지원/정책

### 제품 방향
- 게임 중심 서비스
- 소개/소식/커뮤니티가 게임을 보조
- 모바일에서는 몰입형 플레이 우선

### 디자인 방향
- 따뜻한 아이보리 바탕
- 민트/살구/라벤더 포인트
- 둥근 카드
- 큰 버튼
- 정돈된 앱 UI 위의 디지털 펫 감성

### 홈 핵심 구조
#### 비로그인 홈
상단바 → 히어로 → 핵심 재미 3카드 → 소식 → 커뮤니티 하이라이트 → 시작 가이드 → 푸터

#### 로그인 홈
상단바 → 내 펫 카드 → 오늘 할 일 → 빠른 메뉴 → 소식 → 커뮤니티 하이라이트
