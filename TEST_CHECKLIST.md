# 테스트 체크리스트

## 🚀 개발 서버 실행 방법

### 1. 환경변수 설정 (Firebase 사용 시)
프로젝트 루트에 `.env` 파일 생성:
```env
REACT_APP_FIREBASE_API_KEY=your_api_key
REACT_APP_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
REACT_APP_FIREBASE_PROJECT_ID=your_project_id
REACT_APP_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
REACT_APP_FIREBASE_APP_ID=your_app_id

# 저장소 타입 (localStorage 또는 firestore)
REACT_APP_STORAGE_TYPE=localStorage
```

### 2. 개발 서버 실행
```bash
cd digimon-tamagotchi-frontend
npm install  # 처음 실행 시
npm start
```

브라우저에서 `http://localhost:3000` 접속

## 🔍 Lazy Update 상세 테스트

### 시나리오 1: 오프라인 후 접속
1. 게임 시작
2. 브라우저를 닫고 10분 대기
3. 다시 접속
4. **확인**: 배고픔이 감소했는지 확인
5. **확인**: `lastSavedAt`이 업데이트되었는지 확인

### 시나리오 2: 액션 전 Lazy Update
1. 게임 시작
2. 브라우저를 닫고 5분 대기
3. 다시 접속
4. 먹이 주기 버튼 클릭
5. **확인**: 먹이 주기 전에 배고픔이 이미 감소했는지 확인
6. **확인**: 먹이 주기 후 스탯이 올바르게 업데이트되는지 확인

### 시나리오 3: 사망 조건 테스트
1. 게임 시작
2. 배고픔을 0으로 만들기 (개발자 모드 또는 시간 조작)
3. `lastHungerZeroAt`을 12시간 전으로 설정
4. Lazy Update 적용
5. **확인**: 디지몬이 사망하는지 확인
