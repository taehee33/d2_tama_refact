# 스프라이트 번호 수정 문서

**작성일:** 2026년 1월 28일  
**문제:** 티라노몬 수면 중 데블몬 스프라이트가 표시되는 문제

## 🔍 문제 분석

### 발견된 문제
- 티라노몬이 수면 중일 때 301, 302 스프라이트 파일이 올바르게 로드됨
- 하지만 해당 스프라이트 파일이 데블몬의 이미지임

### 스프라이트 번호 비교

**레거시 데이터 (`nonuse/digimonData.js`):**
- 티라노몬: `startNumber: 285`
- 데블몬: `startNumber: 300`

**현재 데이터 (`v1/digimons.js`):**
- 티라노몬: `sprite: 290` (TODO: Check actual sprite 주석 있음)
- 데블몬: `sprite: 300`

### 수면 프레임 계산

**현재 코드 (290 기준):**
- 티라노몬 수면 프레임: 290 + 11 = **301**, 290 + 12 = **302**
- 데블몬 수면 프레임: 300 + 11 = 311, 300 + 12 = 312

**레거시 데이터 (285 기준):**
- 티라노몬 수면 프레임: 285 + 11 = **296**, 285 + 12 = **297**
- 데블몬 수면 프레임: 300 + 11 = 311, 300 + 12 = 312

## 💡 해결 방안

### 수정 내용
티라노몬의 스프라이트 번호를 **290에서 285로 수정**

**이유:**
1. 레거시 데이터와 일치
2. 301, 302가 데블몬 이미지로 보이는 현상과 일치
   - 301이 데블몬의 기본 스프라이트(300)와 매우 가까움
   - 실제 스프라이트 파일 `/images/301.png`가 데블몬 이미지일 가능성

### 수정 후 예상 결과
- 티라노몬 수면 프레임: 296, 297
- 데블몬 수면 프레임: 311, 312
- 스프라이트 번호 충돌 해소

## ⚠️ 주의사항

만약 285로 수정해도 문제가 지속되면:
1. 실제 스프라이트 파일 확인 필요 (`/images/296.png`, `/images/297.png`가 티라노몬 이미지인지)
2. 다른 디지몬의 스프라이트 번호도 확인 필요
3. 스프라이트 시트의 실제 구조 확인 필요
